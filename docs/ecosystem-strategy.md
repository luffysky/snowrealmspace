## 0. 一句話總結

這不是五個獨立產品，而是一個 **漏斗 → 平台 → 分發 → IP** 的整體：

```
Insight（漏斗/CRM）→ 毛行天下（IP/現金流）
                    ↘         ↗
              AI 島（流量引擎 + Z幣 + Pro）
                    ↘         ↗
   SnowRealm Space（情感/高端）  YukiBoard（每日分發觸點）
            —— 全部共用一套 AI 基建、一個身份、一個 Z幣錢包 ——
```

單獨看每個都在紅海裡；**綁成生態**才有別人抄不走的護城河。

## 1. 五個資產盤點

| 專案 | 定位 | 現況 | 技術 | 生態角色 |
|---|---|---|---|---|
| **毛行天下** | 寵物內容 + 電商，有 IP 角色（招財🐹） | 已有電商 | Express + React/Vite | 消費品牌 / IP / 現金流 |
| **Insight Engine** | 問卷→CRM→ERP 洞察 | 導流工具 | Next.js + Supabase | 漏斗頂端 / 數據中樞 |
| **AI 島 v3** | 免費課程 + 遊戲化 + AI 導師 + 創作者島 + Z幣/Pro | **已變現** | Next.js 15 + Supabase | 旗艦 / 流量引擎 / AI 基建範本 |
| **SnowRealm Space** | 私人空間、禮物、隨時間長大 | 未變現 | Next.js 15 + Supabase + Zeabur | 情感 / 高端 / 隱私 AI |
| **YukiBoard** | Android 注音鍵盤 + Whisper 語音 | 未變現 | Kotlin / Android + whisper server | 分發 / 每日高頻觸點 |

**關鍵觀察：AI 島與 Space 各自獨立實作了幾乎一樣的 AI 路由器（候選鏈＋熔斷器＋免費優先）。** 這是最大的重複，也是最大的整合槓桿。

## 2. 生態級商業模式（把五個綁起來賺錢）

### 2.1 三層收入結構

1. **實體/電商現金流** —— 毛行天下賣寵物用品。已經在跑，是最短的變現路徑。
2. **訂閱：SnowRealm Pro（跨產品一張訂閱）** —— AI 島已有 Pro，把它升級成**生態通用**：一張訂閱解鎖 AI 島付費模型、Space 進階 AI/儲存/多空間、YukiBoard 語音分鐘數。訂閱綁定越多產品，流失率越低。
3. **虛擬貨幣：Z幣（生態通用）** —— 在 AI 島「免費賺」（學習/簽到/成就），在任何產品「花」（AI credits、送禮、換寵物用品折扣）。**在一個產品賺、在另一個產品花 = 跨產品留存飛輪。**

### 2.2 為什麼守得住 —— 免費優先的共用 AI 基建

大部分 AI 情感/工具 app 都在燒 GPT-4 的錢，撐不起慷慨免費層。
我們把 AI 島的 `ai-router` 與 Space 的 `@snowrealm/ai-core` **合併成一個語言無關的 HTTP AI 服務**：
- Zeabur 上**一把 master key**，各產品呼叫同一個端點
- 免費模型優先（CF Workers AI / Groq / Gemini free）→ 邊際成本趨近零
- 共用 quota、成本儀表板、熔斷器

→ 這讓「慷慨免費層」變成可持續，是別人抄不走的成本護城河。

## 3. 兩兩怎麼搭（磨合與整合）

### 3.1 SnowRealm Space × AI 島
- **出去 vs 回家**：AI 島是「公共學習/創作島」，Space 是「你的私人空間」。你在 AI 島的成就/作品集/筆記，**回流成 Space 的個人時間軸與記憶**。
- **免費 vs 高端**：AI 島免費層負責獲客；Space 的乾淨 ai-core + 隱私故事當**付費/高端層**。
- **創作雙向**：創作者島的 AI 內容 → 存進 Space；Space 的 Creative Core 專案 → 發佈到 AI 島社群。
- **共用**：登入、Z幣錢包、Pro 權益。

### 3.2 SnowRealm Space × 毛行天下 ★ 最被低估的一條
- 毛行天下有 **IP 角色 + 精準的養寵受眾**。Space 出一個「**寵物版**」——一個記住毛孩的私人空間（照片、時間軸、里程碑、記得你毛孩的 AI）。
- **情感禮物 × 寵物羈絆/思念 = 極強付費意願**，而毛行天下的受眾正是買家。
- 電商反向：毛行天下把「送一個 Space」當商品賣；Space 內推薦毛行天下的寵物用品（分潤）。

### 3.3 Insight Engine × 全部
- 它就是**通用漏斗**。一個測驗「哪個 SnowRealm 產品適合你 / 什麼寵物空間適合你」→ 收 email → CRM → 養成 → 導去 AI 島註冊 / Space 送禮 / 毛行天下下單。
- 它的 ERP 洞察層 → 升級成**全生態共用分析**。

### 3.4 YukiBoard × 全部 ★ 分發王牌
- 鍵盤是**永遠在線**的觸點。從鍵盤就能召喚 AI 島綠寶導師 / Space 的 agent / 毛行天下內容。
- **語音日記殺手級閉環**：鍵盤語音 → 直接寫進 Space 的每日循環。隨處捕捉（鍵盤）→ 沉澱進你的私人空間。
- 變現：鍵盤免費獲客，語音/AI 進階功能收進生態 Pro。

## 4. 整合優先序（技術磨合，高槓桿在前）

1. **抽出一個共用 AI 服務**（語言無關 HTTP API，因為毛行天下是 Express、YukiBoard 是 Android，不能用共用 npm 套件）。消除 AI 島/Space 的重複，一把 master key。**最高槓桿。**
2. **單一登入 + 共用身份**（snowrealm.pet 各子網域 SSO）。
3. **Z幣錢包 + Pro 權益服務**（AI 島已有雛形，抽成獨立服務）。
4. **Insight 當共用 CRM/漏斗**，把各產品註冊都埋點串進去。
5. **共用設計系統**（Space 的 `--sr-*` token 已是嚴格 token 系統，可當生態設計語言）。

---

## 5. 誠實的風險

- **五個產品對小團隊太多**，最大風險是攤太薄。排序要狠：
  - **AI 島**（有營收＋受眾）= 引擎
  - **毛行天下**（有電商）= 現金流
  - 其餘三個 = 槓桿/差異化，**別一次做完所有整合**
- **技術異質**：MaoTravelBlog 是 Express、YukiBoard 是 Kotlin/Android、其餘 Next.js。共用層必須走 HTTP API，不能靠共用程式碼套件。
- **Z幣/金流合規**：虛擬貨幣跨產品流通牽涉金流與稅務，要先確認法遵再擴大。

## 6. 建議的第一步（90 天）

1. 把 `ai-core` 抽成獨立 HTTP 服務，先讓 AI 島與 Space 共用（驗證「一把 key、免費優先」在跨產品成立）。

4. 訂閱統一成 SnowRealm Pro，先涵蓋 AI 島 + Space。
